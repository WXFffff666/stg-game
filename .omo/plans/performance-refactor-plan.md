# STG 性能重构计划

## 诊断结论

**卡顿主因是客户端 JavaScript/Canvas 性能，不是 Cloudflare Pages 平台限制。**

CF Pages 仅托管静态文件（HTML/JS/CSS），游戏逻辑在用户浏览器执行。平台提供全球 CDN 分发，对纯静态 Canvas 游戏无 CPU 限制。卡顿来自：

- 导弹发射时重复 O(n) 寻敌
- 玩家弹×敌人暴力碰撞 O(B×E)
- 爆炸粒子未走对象池导致 GC 尖峰
- 导弹拖尾粒子洪峰
- HUD 每帧全量 DOM 更新 + backdrop-filter 毛玻璃

## 已实施优化（v18）

1. 导弹发射复用 `HomingTargetManager`，消除重复全量寻敌
2. `WeaponManager._findNearestEnemy` 走共享寻敌 + 空间网格
3. 玩家弹碰撞改用 4×4 空间网格窄相位
4. 导弹爆炸改用 `ParticleSystem.spawn` 对象池（6 粒子 vs 14 临时对象）
5. 导弹拖尾按特效质量/低性能模式动态降频
6. 低特效质量关闭 `lighter` 混合模式
7. HUD 降频更新（正常 3 帧/次，低性能 5 帧/次）
8. 碰撞查询复用 `_nearbyScratch` 缓冲，零分配
9. 移动端禁用 HUD `backdrop-filter`
10. SW 补全 faction 模块预缓存 + `sw.js` no-store 头
11. 脚本版本统一 bump 至 v=18
